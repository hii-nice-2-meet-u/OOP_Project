[✅] Abstract Class: สร้างอย่างน้อย 1 คลาส (นำไปใช้ให้สมเหตุสมผล)

[✅] Inheritance 2 ระดับชั้นขึ้นไป (Person -> Noncustomer -> Staff)

[✅] Polymorphism: มีการทำ Overriding หรือ Overloading อย่างน้อย 1 จุด

[✅] Encapsulation: Attribute (ตัวแปร) ทุกตัวต้องเป็น private ทั้งหมด

[✅] Clean Code: ลบ Getter/Setter ตัวที่ไม่ได้ถูกเรียกใช้งานทิ้งไป

📦 หมวดที่ 2: การออกแบบคลาส (รวมทั้งหมดต้องเกิน 10 คลาสที่มี Logic รัดกุม)
[✅] Class มากกว่า 10 คลาสขึ้นไป โดยไม่นับ Class ที่ Inherit ที่ไม่มี Logic ข้างในมากพอ

[✅] Entity Class (>= 3 คลาส): คลาสข้อมูลพื้นฐาน (เช่น User, Room, Product)

[✅] Transaction Class (>= 2 คลาส): คลาสเก็บประวัติการทำรายการ (เช่น Booking, Order, Payment)

[✅] State Tracking Class (>= 1 คลาส): คลาสที่มีการเปลี่ยนสถานะ (เช่น Pending -> Paid -> Completed) โดยต้องมีอย่างน้อย 3 สถานะ

[✅] Controller Class (1 คลาสเท่านั้น): คลาสศูนย์กลางที่ใช้จัดการระบบและเชื่อมโยงคลาสอื่นๆ

💼 หมวดที่ 3: กฎธุรกิจ (Business Logic)
[✅] Limit Rules: สร้างกฎจำกัด "จำนวน" หรือ "เวลา" อย่างน้อย 2 กฎ (เช่น จองได้ไม่เกิน 5 ชิ้น, หมดเขตภายใน 24 ชม.)

[✅] Cost Components: ระบบคำนวณเงินต้องมีส่วนประกอบอย่างน้อย 4 รายการ (ราคาโต๊ะ + ค่าอหาร - ค่าส่วนลด + ค่าทำเกมพัง)

[✅] Promotion: ทำระบบ สิทธิ์ / คูปอง / หรือส่วนลด อย่างน้อย 1 ประเภท

🛡️ หมวดที่ 4: ความเสถียร (Error Handling)
[✅] Input Validation: ดักจับและตรวจสอบข้อมูลขาเข้า (Input) ทุกจุด อย่าให้โปรแกรมพังเพราะ User พิมพ์ผิด

[✅] Try-Except: ใช้ try-except ดัก Error อย่างน้อย 2 จุด (เช่น ตอนแปลง String เป็น Int หรือตอนดึงข้อมูลที่ไม่มีอยู่จริง)
