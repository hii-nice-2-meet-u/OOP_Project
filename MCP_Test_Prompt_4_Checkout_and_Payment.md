คุณคือผู้ช่วยทดสอบระบบ Board Game Cafe ผ่าน MCP Tools กรุณาทำการทดสอบ **ระบบคิดเงินและ Check-out** ตามขั้นตอนด้านล่างนี้ โดยรายงานผลลัพธ์ Expected VS Actual

### 1. การเตรียมข้อมูล (Setup)
- ใช้ `get_all_cafe_branches` เลือกสาขา (เช่น Ladkrabang)
- ตรวจสอบ Owner/Manager ID และใช้ช่วยใช้คำสั่ง `add_spent` สร้างสมาชิกระดับ "Platinum" (สมมติชื่อ "Rich Member")
- สร้างลูกค้าระดับธรรมดา (Bronze) มา 1 คน (ชื่อ "Normal Member")
- ใช้ `check_in` แบบ Walk-in สองคนรวดให้เกิด 1 Play Session
- ขอ `join_session` ให้คนที่มี Tier สูงสุดอยู่ในโต๊ะด้วย (Platinum)

### 2. Test Order Placement (สั่งอาหาร+ยืมเกมไว้คิดเงิน)
- ใช้ `take_order` เพื่อสั่งอาหารที่มีราคาสูงที่สุดในระบบ 1 รายการ 
- ใช้ `borrow_board_game` เพื่อยืมบอร์ดเกม 1 กล่อง
- จากข้อข้างบน ลองขอจำลองสถานการณ์ให้เกมที่ยืมถูกตีกลับไปเป็น `MAINTENANCE` หรือถ้า Tool บังคับให้ออกสถานะไม่ได้ให้ทึกทักเอาว่ากล่องนั้นพัง

### 3. Test Discount Calculation (คำนวณส่วนลด Tier ในโต๊ะเดียวกัน)
- ในโต๊ะนี้มีคน 2 ระดับคือ Bronze (0%) และ Platinum (25%)
- ตอนเรียกใช้งานคำสั่งชำระเงิน `check_out` ให้ลองเปรียบเทียบบิลที่ออกดูว่า ระบบจะมอบส่วนลดตาม Tier ที่สูงสุด (25%) หรือไม่ (Expected: ต้องใช้ส่วนลดสูงสุด)

### 4. Test Insufficient Payment (แกล้งจ่ายเงินไม่พอ)
- ให้ใช้คำสั่ง `check_out` แต่ระบุ `method_type="cash"` พร้อมกับพยายามแนบยอดเงิน (ถ้า Tool รองรับการกรอกเงิน หรือถ้าไม่ได้ ให้วิเคราะห์ฟังก์ชันจ่ายเงินว่ามีดักเงินขาดหรือไม่)
- **คาดหวัง:** จำลองว่าจ่ายแค่ 100 บาทกับบิล 500 บาท หากได้ต้องโยน Failed ออกมา

### 5. Check Output and State Updates (จบคืนโต๊ะ)
- บันทึกยอดเงินรวมที่จ่ายหลังหักส่วนลด (และ/หรือค่าปรับบอร์ดเกม)
- หลังจ่ายจบ ใช้ `get_person_by_type` (สำหรับ Member) เพื่อดึงยอด `total_spent` กลับมาดูว่า ลูกค้าในโต๊ะได้รับการปัดยอดเข้าบัญชีเพื่อสะสม Tier ครบถ้วนไหม
- เช็คว่าโต๊ะกลับมามีสถานะ `AVAILABLE` ไหม

สรุปผลสิ่งที่เกิดขึ้นเทียบกับ Expected
