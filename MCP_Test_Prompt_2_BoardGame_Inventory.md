คุณคือผู้ช่วยทดสอบระบบ Board Game Cafe ผ่าน MCP Tools กรุณาทำการทดสอบระบบ **คลังบอร์ดเกม (Inventory)** ตามขั้นตอนด้านล่างนี้ และรายงานผล Expected VS Actual

### 1. การเตรียมข้อมูล (Setup)
- ใช้ `get_all_cafe_branches` เลือกสาขา และ `get_branch_board_games` ในสาขานั้นเพื่อดึงรายการบอร์ดเกมมา 3 เกม (สมมติว่าเป็น เกม X, เกม Y, เกม Z ทุกเกมต้องสถานะ `AVAILABLE`)
- ใช้ `check_in` รับลูกค้าเข้าสาขาแบบ Walk-in 2 คน เพื่อให้ได้ Play Session ID มา 2 โต๊ะ (โต๊ะ 1 กับ โต๊ะ 2)

### 2. Test Over-borrowing (ยืมเกินโควตา 2 เกมต่อ Session)
- โต๊ะ 1 ใช้ `borrow_board_game` ขอยืม เกม X (ต้องสำเร็จ)
- โต๊ะ 1 ขอยืม เกม Y เพิ่ม (ต้องสำเร็จ)
- โต๊ะ 1 ขอยืม เกม Z เป็นกล่องที่ 3 
- **คาดหวัง:** ต้องล้มเหลว (Failed) เพราะกฎคือ 1 Play Session ยืมเกมได้สูงสุด 2 กล่องพร้อมกัน

### 3. Test Cross-borrowing (โต๊ะอื่นแย่งเกมที่ถูกยืมอยู่)
- ต่อเนื่องจากข้อ 2 (เกม X ตอนนี้สถานะน่าจะเป็น `IN_USE` จากโต๊ะ 1 ไปแล้ว)
- ให้ โต๊ะ 2 (Play Session 2) ลอง `borrow_board_game` เกม X 
- **คาดหวัง:** ล้มเหลว (Failed) เพราะเกม X ไม่ว่าง (Not Available)

### 4. Test Game Return & Random Maintenance Limit (การคืนและโอกาสของพัง)
- ให้ โต๊ะ 1 คืนเกม (`return_board_game`) ทีละเกม (เกม X และ เกม Y)
- สังเกตสถานะของเกม X และ Y หลังคืนสำเร็จ (มันมีโอกาส 10% ตามโค้ดที่จะกลายเป็น `MAINTENANCE`)
- **ถ้าโชคดีเจอเกมพัง:** ให้โต๊ะ 2 ลองยืมเกมที่พัง (ต้อง Failed)
- **คาดหวัง:** ต้องคืนได้เรียบร้อย ระบบปลดล็อคให้กล่องนั้นกลับไปให้คนอื่นยืมต่อได้ (หรือขึ้นซ่อม)

ทำเสร็จแล้วสรุปผล Expected VS Actual ในรูปแบบตารางครับ
