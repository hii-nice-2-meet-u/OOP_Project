# MCP Reservation Rule Test Prompt

ก็อปปี้ Prompt ด้านล่างนี้ไปวางใน Claude Desktop ที่เชื่อมต่อกับ MCP Server `BoardGameCafe` แล้ว เพื่อให้ Claude ทำการทดสอบกฎการจอง (Reservation Rules) ทั้งหมดทีละขั้นตอน

คุณคือผู้ช่วยทดสอบระบบ Board Game Cafe ผ่าน MCP Tools กรุณาทำการรันสคริปต์ทดสอบระบบการจอง (Reservation) ตามขั้นตอนด้านล่างนี้อย่างเคร่งครัด โดยทำทีละหัวข้อและรายงานผลลัพธ์ว่าสำเร็จ (Success) หรือล้มเหลว (Failed ตรงตามที่คาดหวังหรือไม่) 

ใช้ Tool ที่มีอยู่เพื่อจำลองสถานการณ์ต่างๆ (ใช้ `make_reservation`, `create_customer_member` ฯลฯ) 
**หมายเหตุ**: เนื่องจากใน MCP Tool การสร้างสมาชิกระดับต่างๆ อาจตายตัวเป็น Bronze ตอนที่สร้างใหม่ ให้ถือว่าทดสอบเท่าที่ระบบ Tool เอื้ออำนวย หรือถ้าให้เจาะจงรายชื่อคนที่มี Tier อยู่แล้วในฐานข้อมูล (Demo) ก็ให้ใช้ชื่อนั้น แต่ถ้าไม่มี ให้สร้างสมาชิกใหม่ขึ้นมาทดสอบ

แต่เพื่อให้ใกล้เคียงกับ Test Case มากที่สุด ช่วยรันคำสั่งทำจองตามเงื่อนไขต่อไปนี้:

### 1. การเตรียมข้อมูล (Setup)
- ตรวจสอบสาขาที่มีอยู่ในระบบ (get_all_cafe_branches) เลือกมา 1 สาขา (เช่น "Ladkrabang" หรือชื่อสาขาที่ได้จากระบบ)
- ตรวจสอบรายชื่อพนักงานที่มีสิทธิ์เป็น Owner หรือ Manager (`get_person_by_type("Owner")` หรือ `get_person_by_type("Manager")`) เพื่อนำ ID มาใช้เป็น Authorizer
- สร้างสมาชิกลูกค้าใหม่ 4 คน ชื่อ "Test Bronze", "Test Silver", "Test Gold", "Test Platinum" โดยตอนสร้างใหม่ทั้งหมดจะเป็นระดับ Bronze
- อัปเกรดระดับสมาชิกของแต่ละคนโดยใช้ `add_spent` (ให้ Authorizer ID ที่ได้มาก่อนหน้าเป็นผู้อนุมัติ):
  - "Test Silver" เพิ่มยอดใช้จ่าย 500
  - "Test Gold" เพิ่มยอดใช้จ่าย 1000
  - "Test Platinum" เพิ่มยอดใช้จ่าย 2000
*สมมติฐานให้ใช้วันเวลาปัจจุบันของระบบในการคำนวณ*

### 2. Test Minimum Lead Time (ต้องจองล่วงหน้า 1 ชั่วโมงขึ้นไป)
- พยายามจองโต๊ะให้ "Test Bronze" ล่วงหน้าจากเวลาปัจจุบันเพียงแค่ 30 นาที เป็นเวลา 2 ชั่วโมง
- **ความคาดหวัง:** ต้องล้มเหลว (Failed) เพราะ Lead Time ไม่ถึง 1 ชั่วโมง

### 3. Test Max Duration (ระยะเวลาจองสูงสุดตามระดับสมาชิก)
ทำการทดสอบการจอง *ในวันพรุ่งนี้* (เพื่อไม่ให้ติดกฎ Lead Time):
- **Bronze:** ให้ "Test Bronze" จองโต๊ะเป็นเวลา 3 ชั่วโมง (คาดหวัง: Failed เพราะเกิน 2 ชม.)
- **Silver:** ให้ "Test Silver" จองโต๊ะเป็นเวลา 4 ชั่วโมง (คาดหวัง: Failed เพราะเกิน 3.5 ชม.)
- **Gold:** ให้ "Test Gold" จองโต๊ะเป็นเวลา 8 ชั่วโมง (คาดหวัง: Failed เพราะเกิน 7 ชม.)
- **Platinum:** ให้ "Test Platinum" จองโต๊ะเป็นเวลา 12 ชั่วโมง (คาดหวัง: Success เพราะได้สูงสุด 12 ชม.)

### 4. Test Advance Booking Limits (วันจองล่วงหน้าสูงสุด)
- ให้ "Test Bronze" พยายามจองโต๊ะล่วงหน้าออกไปอีก 7 วันนับจากวันนี้
- **ความคาดหวัง:** ต้องล้มเหลว (Failed) เพราะ Bronze จองล่วงหน้าได้แค่ 5 วัน

### 5. Test Active Quota (โควตาการจองค้างในระบบ)
- ให้ "Test Bronze" (มี Quota จองล่วงหน้าได้ 1 รายการ) ทำการจองโต๊ะล่วงหน้าในอีก 2 วันข้างหน้า (คาดหวัง: Success)
- ให้ "Test Bronze" คนเดิม ทำการจองโต๊ะล่วงหน้าอีกครั้งในอีก 3 วันข้างหน้า (โดยที่ยังไม่ยกเลิกของเดิม)
- **ความคาดหวัง:** ต้องล้มเหลว (Failed) เพราะโควตาสำหรับ Bronze คือ 1 รายการเท่านั้น

### 6. Test Double Booking (ห้ามจองซ้อนเวลาบนโต๊ะเดียวกัน)
- สร้างลูกค้า "Test DB1" และ "Test DB2"
- ให้ "Test DB1" จองโต๊ะในสาขา วันพรุ่งนี้ เวลา 14:00 - 16:00
- เช็คว่า "Test DB1" ได้โต๊ะหมายเลขอะไร (สมมติว่าเป็น โต๊ะ A)
- ให้ "Test DB2" ทำการจองในสาขาเดียวกัน วันพรุ่งนี้ เวลา 15:00 - 17:00 แบบเจาะจงโต๊ะ (ถ้า tool รองรับ) หรือจองเพื่อดูว่าระบบจัดสรรโต๊ะ A ให้ซ้ำหรือไม่/ปฏิเสธหรือไม่ 
- **ความคาดหวัง:** ระบบต้องไม่ให้ "Test DB2" จองโต๊ะ A ซ้อนในเวลา 15:00 - 16:00 (Overlap Time)

หลังจากทำการทดสอบครบทุกข้อแล้ว กรุณาสรุปผลลัพธ์ทั้งหมดเป็นตารางเปรียบเทียบว่า **สิ่งที่คาดหวัง (Expected)** กับ **สิ่งที่เกิดขึ้นจริง (Actual)** ตรงกันหรือไม่
